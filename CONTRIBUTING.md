# Contribuir a Eclipse

## Estilo de trabajo

- Cambios pequeños y revisables.
- Seguridad antes que automatización llamativa.
- Documentar decisiones importantes en `docs/adr/`.
- No agregar dependencias pesadas sin justificar.
- No subir secretos, tokens, screenshots personales ni audio privado.

## Flujo sugerido

1. Crear issue o tarea.
2. Implementar una capacidad mínima.
3. Agregar pruebas o checklist de verificación.
4. Actualizar documentación.
5. Revisar riesgos de seguridad.

## Criterios para aceptar una nueva tool

- Tiene propósito claro.
- Declara nivel de riesgo.
- Tiene límites/allowlist.
- Tiene logs.
- Pide confirmación si corresponde.
- Tiene modo de prueba seguro.
